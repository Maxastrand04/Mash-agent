# 3.10 — Mash Behaviour Update (list-flow fixes + move/copy missing-destination + bare-name extension)

Fix four bugs surfaced while exercising the 3.9 list/open/cat flow, and apply two design refinements from `mash_behavior.md`.

**Goal:** After this plan:
- Move / copy with no destination given (including when routed from the list action menu) prompts a current-directory menu instead of fuzzy-searching for the literal token `unknown`.
- Typing a folder/file name during list flow fuzzy-matches it against the tree: folder → switch listing scope; file → narrow the listing.
- Listing a subdirectory shows the header as `Mash listing ./<subdir>:` (relative), not the absolute path.
- The current directory always appears as the last entry of every list menu, suffixed `(Current directory)`.
- Creating a file via the list `add` action with a single-token name (e.g. `apple`) prompts for an extension before running `touch`.
- Rename routed from the list action menu applies the same extension-reconciliation prompts as the direct CLI rename.
