# 1.0 — Core Agent

Sets up the Python package and implements the full `mash` CLI: parse args → call local LLM → confirm with user → execute.

**Goal:** A working `mash` command installable via `pip install -e .` that takes natural language, generates a bash command via a local Ollama model, and runs it after user confirmation.

## Claude Instructions

- **Model:** Always use `qwen2.5-coder:7b` via `ollama.chat()`. Do not make this configurable.
- **System prompt:** Must instruct the model to output a single bare bash command — no markdown, no explanation, no extra text. Strip the response before returning.
- **Confirmation default:** The `[y/N]` prompt defaults to N — anything other than lowercase `y` cancels.
- **No shell history or context:** Each invocation is stateless. Do not pass previous commands or output to the model.
- **Entry point:** `mash.main:run` — `run()` reads `sys.argv[1:]`, so no argparse needed for v1.
- **Error cases:** Exit with a usage hint if no args given. Exit with a message if the LLM returns an empty string.
