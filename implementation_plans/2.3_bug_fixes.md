## Phase 1 — Fix B2: filter non-filename words before resolving

**File:** `mash/main.py`

Keep `resolver.py` as a pure fuzzy matcher. Add the filter at the call site in `main.py`.

Add `_STOP_WORDS` and `_is_filename_candidate` above `run()`:

```python
_STOP_WORDS = {
    "move", "copy", "rename", "delete", "remove", "list", "create", "make",
    "open", "show", "get", "put", "add", "find",
    "to", "the", "a", "an", "in", "into", "from", "of", "and", "or",
    "inside", "under", "called", "named",
    "file", "files", "folder", "folders", "directory", "dir",
    "all", "new", "old", "my", "its", "current",
}

def _is_filename_candidate(word: str) -> bool:
    # A dot is a reliable filename signal regardless of length or stop-word status
    return "." in word or (len(word) >= 5 and word.lower() not in _STOP_WORDS)
```

## Phase 2 — Fix B3: multi-candidate return + source disambiguation menu

### 2a — resolver.py: return all matches, track all paths per stem

**File:** `mash/resolver.py`

Two changes:
1. `stems` becomes `dict[str, list[str]]` — stores every path that shares a stem, not just the first.
2. New function `resolve_paths(tree, term) -> list[str]` returns all matching paths (up to 5 close matches × all paths per stem), deduplicated. Old `resolve_path` can be removed.

```python
import difflib
from pathlib import Path


def resolve_paths(tree: str, term: str) -> list[str]:
    paths = [line.strip() for line in tree.splitlines() if line.strip()]
    stems: dict[str, list[str]] = {}
    for p in paths:
        stem = Path(p).stem.lower()
        stems.setdefault(stem, []).append(p)
    matches = difflib.get_close_matches(term.lower(), stems.keys(), n=5, cutoff=0.6)
    result = []
    for m in matches:
        for p in stems[m]:
            if p not in result:
                result.append(p)
    return result
```

## Phase 4 — Filename format selection for new files

When the user's intent is to **create** a new file and the filename is multi-word (i.e. more than one word appears after "called" or "named" in the prompt), intercept before calling the LLM and present a format menu:

```
What filename format do you want?

  1. mash_testing.py   (snake_case)
  2. mash-testing.py   (kebab-case)
  3. mashTesting.py    (camelCase)

Select format [1-3] (Enter to cancel):
```

Implementation notes:
- Add `_detect_new_filename(args) -> tuple[str, str] | None` in `main.py`. Returns `(raw_name, extension)` if the prompt looks like a create-with-name request, else `None`. Detection: first word is "create"/"make", and a word "called"/"named" is present followed by at least one more word. Extension comes from the first token after "called"/"named" if it contains a `.`, otherwise infer from context words ("python" → `.py`, "text"/"txt" → `.txt`, "markdown"/"md" → `.md`; default to no extension).
- Add `_format_filename(raw_name, extension) -> list[str]` that returns the three formatted variants.
- Show the menu, get selection, then inject `Resolved filename: <chosen_name>` into the LLM prompt so the model uses it exactly.
- `--yes` → auto-select snake_case (#1), print `Auto-selected filename: <name>`.
- `--dry-run` → print all options, auto-select #1, print `[dry-run] auto-selected filename: <name>`.

## Claude Instructions

- **Order:** 1 → 2a → 2b → 3a → 3b → 4 → 5 → 6. `resolver.py` changes (2a, 3a) before `main.py` changes that import them.
- **Scope:** Only `mash/resolver.py` and `mash/main.py`. No other files.
- **`_is_filename_candidate` dot rule:** A word containing `.` is always a candidate even if short or in stop words. Catches `main.py`, `data.csv`, `notes.txt`.
- **`resolve_dirs` directory detection:** `find` output includes directories as paths with no suffix (e.g. `./reports`, `./docs`). Use `Path(p).suffix == ""` to identify them. Do not include `.` (current dir) itself.
- **Destination token extraction:** scan `args` left-to-right for the first occurrence of "to", "into", or "inside" and take the next token as the destination word. If none found, skip destination resolution.
- **`disambiguate` reuse:** the same function handles source files, destination dirs, and (optionally) other future menus — the `label` parameter ("files" / "folder paths") controls the header wording.
- **Filename format menu is pre-LLM:** resolve the filename before building the LLM prompt so the model receives `Resolved filename: hello_world.py` and outputs `touch ./hello_world.py`.
- **B1 prompt fix:** the system prompt guard is a safety net. The primary B1 fix is the resolved destination being passed explicitly.
