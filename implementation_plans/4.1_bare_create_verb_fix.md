## Claude Instructions

- **Architecture:** `parse_intent` determines `verb` and `Intent` flags; `create_flow` owns all disambiguation (file vs folder, extension picking, destination selection). The parser should never try to resolve ambiguity that belongs to the flow. When the first word is "create"/"make", the verb is always "create" — the flow decides the rest.
- **The fix:** In `parse_intent` (line ~218), the verb-assignment chain is `if is_rename → elif is_create_file or is_create_folder → else`. Insert `elif is_create_form: verb = "create"` between the create-flag branch and the else block. This catches bare creates where neither flag is set.
- **Constraints:** Do not change `create_flow`, `_ask_create_kind`, `_pick_extension_auto`, or any other module. Do not add new features or error handling. Do not change any prompt text or menu wording. Do not touch files outside `mash/intent.py` (except `bugs_found.md` for reporting).
- **Order dependency:** Phase 0 (read-only confirmation) before Phase 1 (edit). Phase 1 before Phase 2 (tests).
- **Testing:** All test commands run from `claude_testing/`. Never `cd` outside it. Use the fixture reset from test plan 3.0 Phase 0 before running tests. Verify the 4 regressions pass and the 2 spot-checks don't regress.
