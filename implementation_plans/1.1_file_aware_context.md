# 1.1 — File-aware directory context

Inject current directory tree into the LLM prompt so file operations resolve to real paths.

**Goal:** When a user asks Mash to move, remove, rename, or otherwise act on a file, the LLM receives a listing of the current directory tree and generates a command using the correct path.

## Claude Instructions

- **Only file:** `mash/main.py` — no other files to touch.
- **`get_directory_context()`:** Use `find . -maxdepth 4 -not -path "*/.*"` to skip hidden files and avoid runaway output in large trees.
- **Prompt injection:** Prepend the tree to the user message as `"Current directory tree:\n{context}\n\nUser request: {prompt}"` — do not put it in the system prompt (keeps system prompt stable and cacheable).
- **System prompt addition:** Add a sentence telling the model to use the directory listing for path resolution and to never go above the current directory (`..`).
- **No other changes:** Confirmation flow, model name, and single-command output requirement stay the same.
- **Testing:** `cd` to a directory with nested files, run `mash move <file> to <subdir>/` and verify the generated command uses the correct relative path.
