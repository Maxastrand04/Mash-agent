# 1.2 — Command templates + Python file resolver

Replace LLM-generated file paths with Python-resolved paths and hardcoded bash templates, so the model only fills in intent — not paths or command syntax.

**Goal:** The LLM receives a pre-resolved file path and a set of concrete bash command templates. It picks the right template and fills in the slots — it never constructs `find`, guesses extensions, or invents paths.

## Phase 2 — Python file resolver

Create `mash/resolver.py` with a function that takes the raw directory tree string and a search term, and returns the best-matching path.

Strategy:
1. Parse tree lines into a list of paths
2. Strip extensions and lowercase for comparison
3. Use `difflib.get_close_matches` for fuzzy matching
4. Return the full original path (with extension) of the best match, or `None`

```python
from pathlib import Path
import difflib

def resolve_path(tree: str, term: str) -> str | None:
    paths = [line.strip() for line in tree.splitlines() if line.strip()]
    stems = {Path(p).stem.lower(): p for p in paths}
    matches = difflib.get_close_matches(term.lower(), stems.keys(), n=1, cutoff=0.6)
    return stems[matches[0]] if matches else None
```

## Claude Instructions

- **Architecture:** Three concerns stay separate — `commands.py` (raw shell commands), `templates.py` (intent → bash template), `resolver.py` (fuzzy file matching). `main.py` only orchestrates.
- **Resolver:** `difflib.get_close_matches` with `cutoff=0.6` is a good starting point. Only match against `Path.stem` (no extension), lowercase.
- **Constraints:** Don't try to extract the search term from the user's sentence in Python — pass the full user prompt to the LLM and let it pick the template. The resolver is only called when a file-like token can be identified, or we pass the full tree and let the LLM reference it.
- **Order dependency:** Phase 1 and 2 are independent and can be done in parallel. Phase 3 depends on both.
- **Testing:** `mash move test into implementation_plans` should produce `mv ./test.pdf ./implementation_plans/` (or similar) with no `find`, no `-type d`, no guessing.
