# 3.1 — Fixes for bugs found in verification run (V1–V4)

Address the four bugs surfaced by [3.0_test_plan.md](../test_plans/3.0_test_plan.md) / `Claude_testing/verification_report.md`. The common root cause is that menu selections (`resolved`, `destination`, `filename`) are passed to the LLM as *hints* in the prompt and the model is free to ignore them. Fix this by post-processing the LLM output rather than relying on the model to honour hints, and tighten resolver scope for create-form prompts.

**Goal:** After this plan, all four FAIL tests from the 3.0 verification run pass. No regressions in Phases 1, 2, 4.4, 5, 6 of that suite.

## Phase 1 — V3: cancel should abort, not silently continue

**File:** `mash/main.py`

In `run()`, after each `disambiguate(...)` call where the user could have cancelled (source menu, destination menu) and after the filename format prompt, distinguish "no candidates / not applicable" from "user cancelled". Cancellation should print `Cancelled.` and `sys.exit(0)` before the LLM call.

Concrete change: `disambiguate()` already returns `None` for both "no candidates" and "user cancelled". Split these by checking *before* calling: if `candidates` was non-empty and the result is `None`, that's a cancel.

```python
if source_candidates:
    resolved = disambiguate(source_candidates, "files", yes, dry_run)
    if resolved is None and not (yes or dry_run):
        print("Cancelled.")
        sys.exit(0)
else:
    resolved = None
```

Same pattern for the destination menu. For the filename menu, treat empty/invalid input as cancel (currently sets `filename = None`).

## Phase 2 — V4: exclude create-form name tokens from source resolution

**File:** `mash/main.py`

In `run()`, before the source-resolution loop, compute the set of word indices that come *after* a `called`/`named` marker (those tokens describe a *new* filename to create, not an existing one to find). Skip those when collecting source candidates.

```python
skip_after_marker = False
filtered_args = []
markers = {"called", "named"}
for w in args:
    if skip_after_marker:
        continue
    if w.lower() in markers:
        skip_after_marker = True
        continue
    filtered_args.append(w)

seen: dict[str, None] = {}
for word in filtered_args:
    if _is_filename_candidate(word):
        for p in resolve_paths(context, word):
            seen[p] = None
```

Only apply the filter when the prompt is a create-form (`args[0]` in `{"create", "make"}`). Otherwise keep current behaviour.

## Phase 3 — V1 & V2: enforce menu selections via post-processing

**File:** `mash/main.py`

The LLM ignores hints. Stop trusting it for these decisions: after `ask_llm(...)` returns, rewrite the command to enforce the menu choices.

### 3a — Enforce `filename`

When the filename menu produced a `filename` (e.g. `mash_testing.py`), the final command must contain `./<filename>` as the *target path*. The raw multi-word form (`mash testing.py`, with or without escaping) must be replaced.

Implementation: build a small `_apply_filename(cmd, raw_name, filename)` that:
1. Replaces any occurrence of the raw multi-word name with or without escape backslashes (`mash testing`, `mash\ testing`) followed by an optional extension with `./<filename>`.
2. As a fallback, if the command does not contain `./<filename>`, append it: for `touch` / `mkdir`, rewrite the last argument to `./<filename>`.

```python
import re

def _apply_filename(cmd: str, raw_name: str, filename: str) -> str:
    # raw_name like "mash testing"; matches "mash testing.py", "mash\ testing.py", or bare "mash testing"
    pattern = re.escape(raw_name).replace(r"\ ", r"\\?\s")
    pattern = rf"\.?/?(?:{pattern})(\.[A-Za-z0-9]+)?"
    new = f"./{filename}"
    if re.search(pattern, cmd):
        return re.sub(pattern, new, cmd, count=1)
    parts = cmd.split()
    if parts and parts[0] in {"touch", "mkdir"}:
        parts[-1] = new
        return " ".join(parts)
    return cmd
```

Wire into `run()` after `ask_llm`:

```python
if filename and new_file:
    raw_name, _ = new_file
    cmd = _apply_filename(cmd, raw_name, filename)
```

### 3b — Enforce `destination`

When `destination` was menu-selected, the command must use it. For `mv`/`cp`, replace the destination argument (last positional) with `<destination>/` (trailing slash so it's unambiguously a directory).

```python
def _apply_destination(cmd: str, destination: str) -> str:
    parts = cmd.split()
    if not parts or parts[0] not in {"mv", "cp"}:
        return cmd
    dest = destination.rstrip("/") + "/"
    # replace the final token (the destination)
    parts[-1] = dest
    return " ".join(parts)
```

Wire in:

```python
if destination:
    cmd = _apply_destination(cmd, destination)
```

Order matters: apply filename first (touch/mkdir), then destination (mv/cp). The two never collide because the create-form and move/copy-form prompts use different verbs.

### 3c — System prompt cleanup

Since enforcement is now in code, soften the system prompt rules from "use them exactly" to "prefer them" — the prompt is still useful but no longer load-bearing. Optional; not strictly required for tests to pass.
