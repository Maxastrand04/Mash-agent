# 3.8 ‚Äî Mash Behaviour Update

Align all interactive prompts in `mash` with the unified UX defined in `mash_behavior.md`: a single disambiguation-menu template, absolute paths everywhere, current-directory annotation, extra delete confirmation, and parity between move/copy flows.

**Goal:** After this plan, every prompt mash shows the user follows the universal template (header ‚Üí numbered options ‚Üí `Select option [1-N], Enter to cancel, type name for other file/folder:`), all file/folder entries are absolute paths, the current directory is suffixed with `(Current directory)`, single-match source/destination still uses the numbered menu, delete asks an extra absolute-path confirmation, and copy supports the expanded trigger words.

## Claude Instructions

- **Source of truth:** [mash_behavior.md](../mash_behavior.md). If implementation and cheat sheet disagree, the cheat sheet wins ‚Äî do not silently diverge. If something in the cheat sheet is ambiguous, ask before improvising.
- **Architecture:** all interactive prompts live in `mash/main.py` (`select_source`, `select_destination`, `confirm_and_run`, and the create-name flow inside `run()`). Centralise the menu rendering in one helper so future tweaks are one-line changes.
- **Absolute paths:** use `os.path.abspath(...)` at *display time only*. Do NOT rewrite resolved paths upstream ‚Äî the LLM, post-processing, and `_apply_*` helpers still receive the relative `./...` paths they expect today. Changing internal representation would break the regex-based post-processing in `main.py`.
- **Current-directory label:** rendered as `<abs cwd>  (Current directory)` (two spaces before the parenthetical). Use a single helper so the label is consistent.
- **Single-match menus:** must still print as numbered menus (`[1]`). This is a deliberate UX choice from the cheat sheet ‚Äî do not "optimise" it back into a yes/no confirm.
- **Constraints:**
  - Don't touch [resolver.py](../mash/resolver.py) ‚Äî fuzzy matching is unchanged.
  - Don't change template strings in [templates.py](../mash/templates.py) except to add copy synonyms if needed.
  - Don't introduce new dependencies.
  - Don't add comments/docstrings beyond what the existing file uses.
  - Don't edit anything in `claude_ignore/`.
- **Order dependency:** Phase 1 (helper) before Phases 2‚Äì4 (callers). Phase 5 (delete) and Phase 6 (copy) are independent. Phase 7 last.
- **Testing:** no automated tests exist for the interactive flow ‚Äî verification is manual via the Phase 7 commands. After each phase, run at least one matching command from Phase 7 and visually diff against the cheat sheet.
- **Editable install warning:** see [CLAUDE.md](../.claude/CLAUDE.md) ‚Äî do not run `pip install -e .` unless the entry point changed; if you must, re-apply the `sys.path.insert` fix in `.venv/bin/mash`.
