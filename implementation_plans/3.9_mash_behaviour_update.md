## Claude Instructions

- **Source of truth:** [mash_behavior.md](../mash_behavior.md). If implementation and cheat sheet disagree, the cheat sheet wins. If something is ambiguous, ask the supervisor before improvising.
- **Testing sandbox:** all manual verification must happen inside `Claude_testing/`. Do not create scratch files elsewhere. After each test, clean up scratch files you created so the directory state stays predictable for the next run. Real `rm` and `mv` will execute against `Claude_testing/` during tests ‚Äî that is fine.
- **Architecture:** keep interactive logic in `mash/main.py`. Reuse the existing `_render_menu`, `_abs`, `_cwd_label`, and the confirm-and-run helper. Add a single `_confirm_action(verb, abs_path, yes, dry_run)` helper so the delete / open / cat confirmations share one implementation.
- **List ‚Üí action menu routing:** prefer in-process routing ‚Äî set `resolved` (and `destination`/`is_create_*` as needed) and continue through the existing `run()` flow rather than re-shelling out to `mash`. Re-shelling loses the dry-run state and breaks `--yes` propagation.
- **Decision (BUSINESS):** bare `mash create drafts` with no extension and no `file`/`folder` word ‚Äî does mash assume folder or ask? **Default for this plan: assume folder** (matches the existing folder-create UX and avoids a guess about extension). If the supervisor wants a clarifying prompt instead, flag this before implementing Phase 1 row 2.
- **Decision (BUSINESS):** under `--yes`/`--dry-run`, the list flow's second (action) menu has no obvious default. **Default for this plan: auto-cancel** (do nothing) since there is no safe single action. If the supervisor wants `--yes` to auto-pick the first action (`go into`/`open`), flag this before implementing Phase 2 row 7.
- **Absolute paths at display time only.** Internal resolved paths remain `./‚Ä¶` so the existing post-processing regexes still work. Convert with `_abs()` exclusively when printing.
- **Constraints:**
  - Do NOT touch [resolver.py](../mash/resolver.py).
  - Do NOT run `pip install -e .` (overwrites the venv entry point ‚Äî see [.claude/CLAUDE.md](../.claude/CLAUDE.md)).
  - Do NOT modify or read anything in `claude_ignore/`.
  - Do NOT add docstrings or multi-line comment blocks; match existing comment density.
  - Single-match menus stay numbered (`[1]`). Do not collapse to yes/no.
- **Order dependency:** Phase 1 (create-detection fix) is independent and can land first. Phases 3 + 4 share the `_confirm_action` helper ‚Äî implement Phase 3 before Phase 4 or factor the helper first. Phase 2 (list) is independent but depends on the action flows being usable when routed into.
