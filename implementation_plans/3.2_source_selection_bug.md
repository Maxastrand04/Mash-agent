# 3.2 — Source selection menu doesn't fire for short filenames

Fix the bug where `mash remove main` and `mash move main into docs` skip the source-disambiguation menu. Root cause is `_is_filename_candidate` rejecting words shorter than 5 chars before they reach the resolver.

**Approach:** Instead of guessing which prompt words *might* be filenames (heuristic), try every non-stop-word against the actual file tree and let `difflib`'s cutoff decide. Words with no close match produce no candidates — no false menus. The length threshold and `_is_filename_candidate` become unnecessary.

**Goal:** Source menu fires whenever a prompt word closely matches a real stem in the tree, regardless of word length. No extra stop word expansion needed; the resolver's `cutoff=0.6` does the filtering.

**Status legend:** ⬜ Not started · 🟡 In progress · ✅ Done

## Bug evidence

### Case 1 — no source menu for ambiguous short name

```
$ mash remove main
Run: rm -rf ./main.py
```

Tree has `./main.py` and `./docs/main.py`. Expected: source menu listing both.

### Case 2 — destination menu fires, source menu doesn't

```
$ mash move main into docs

Mash found multiple candidate folder paths:
  1. ./archive/docs
  2. ./docs
Select folder [1-2] (Enter to cancel): 1

Run: mv ./main.py ./archive/docs/
```

Source menu for `main` was expected too.

---

## Root cause

[mash/main.py:53-54](../mash/main.py):

```python
def _is_filename_candidate(word: str) -> bool:
    return "." in word or (len(word) >= 5 and word.lower() not in _STOP_WORDS)
```

`"main"` (4 chars, no dot) → `False`. The resolver is never called for it, so no source menu fires. The LLM still picks a path from the tree on its own — which is why a command is generated — but no disambiguation happens.

The `>= 5` threshold was added to filter out short verbs/prepositions, but it also cuts out valid 3–4 char filenames. The smarter filter is the resolver itself: `difflib.get_close_matches(..., cutoff=0.6)` already returns nothing for words that don't resemble any tree stem.

---

## Phase 1 — Replace the length threshold with tree-driven matching

**File:** `mash/main.py`

Remove `_is_filename_candidate` entirely. In `run()`, try every word that isn't a stop word against `resolve_paths`. The resolver's 0.6 cutoff handles false positives — "move", "from", "for" won't match any real stem so they produce no candidates.

```python
# Before (heuristic pre-filter):
for word in filtered_args:
    if _is_filename_candidate(word):
        for p in resolve_paths(context, word):
            seen[p] = None

# After (tree-driven):
for word in filtered_args:
    if word.lower() not in _STOP_WORDS:
        for p in resolve_paths(context, word):
            seen[p] = None
```

`_STOP_WORDS` is kept as a performance optimisation — it skips words that are guaranteed not to be filenames (command verbs, prepositions). No new entries needed; the resolver filters everything else.

The dot rule (`"." in word`) is no longer needed as a special case because dotted words are short-circuited as stop-word misses anyway, and `main.py` resolves correctly via its stem `main`.

## Phase 2 — Exclude destination token from source resolution

**File:** `mash/main.py`

With a lower threshold, the destination word (e.g. `docs` after `into`) would now also resolve as a source candidate, causing a double menu. Exclude the destination token's argument index from the source loop.

Add a sibling to `_detect_destination` that returns the index instead of the value:

```python
def _detect_destination_idx(args: list[str]) -> int | None:
    markers = {"to", "into", "inside"}
    for i, word in enumerate(args):
        if word.lower() in markers and i + 1 < len(args):
            return i + 1
    return None
```

In `run()`, compute `dest_idx` before the source loop and skip that index:

```python
dest_idx = _detect_destination_idx(args)

# ...inside the filter loop:
for i, w in enumerate(args):
    if skip:
        continue
    if i == dest_idx:
        continue   # destination token — handled separately
    if is_create_form and w.lower() in {"called", "named"}:
        skip = True
        continue
    filtered_args.append(w)
```

This skips only the single destination-token occurrence, not all instances of that word — so `move docs to docs_archive` still resolves `docs` as a source.

## Claude Instructions

- **Order:** Phase 1 → Phase 2 → Phase 3. Phase 1 alone is sufficient for Case 1 (single-dir match); Phase 2 is required for Case 2 (destination double-fire).
- **Scope:** Only `mash/main.py`. Do not touch `resolver.py`, `templates.py`, `commands.py`, or anything outside `mash/` and `Claude_testing/`.
- **Keep `_STOP_WORDS`.** It's a performance guard, not a correctness filter. It prevents redundant resolver calls for known non-filename words. Do not expand it to cover short words that difflib would reject anyway.
- **Delete `_is_filename_candidate`.** It has no callers after Phase 1. Don't leave it as dead code.
- **Derive `dest_token` from `dest_idx`:** after Phase 2, `run()` can compute `dest_token = args[dest_idx]` instead of calling `_detect_destination` separately. Remove `_detect_destination` if it has no other callers; keep it if it's still used.
- **Test 3.5 nuance:** `rename main.py to app.py` — `main.py` has a dot so it was always a candidate. `app.py` is the destination token (word after `to`) and must be excluded from source resolution; otherwise `app.py` would (weakly) fuzzy-match `main.py` and add it as a second source candidate.
- **Test 3.6 sanity check:** short words like `data`, `the`, `reports`, `folder` are either stop words or will not match real stems closely enough. Verify no unexpected source menus appear for clean move prompts.
