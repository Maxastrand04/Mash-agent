# 4.3 — [feature] unify mash architecture: OOP, full names, exceptions (#3)

Restructure `mash/**` so every file is one class with full descriptive names and exception-based alternate flows, with automated enforcement for OOP-only and sentinel-return rules.

**Goal:** A contributor opening any file in `mash/**` finds exactly one class, predictable in shape, with self-explanatory names and `try`/`except` over named exception classes instead of chained sentinel-`None` checks. CLI behaviour is byte-identical to today, rewritten unit tests achieve 100% branch coverage (with documented exclusions), and integration tests assert per-intent command output against a pre-refactor golden capture.

**Status legend:**  ⬜ Not started · 🟡 In progress · ✅ Done

---

## Acceptance criteria

- **AC-1 — OOP only, one class per file** — Every `.py` file in `mash/**` (excluding `__init__.py`) defines exactly one class, and that class owns all behavior + values for the file. No loose top-level function modules; no module-level `def`. `@dataclass` records and module-level assignment re-exports are allowed.
  *Verify by:* `scripts/check_conventions.py` AST-scans `mash/**/*.py`, reports zero module-level `FunctionDef`/`AsyncFunctionDef`, and exits 0.

- **AC-2 — Full descriptive names (documented + reviewed)** — All variable, parameter, function, method, and class names are full descriptive words. No abbreviations: `ext`→`extension`, `dest`→`destination`, `cmd`→`command`, `src`→`source`, `disambig`→`disambiguator`, `dir`→`directory`, etc. Loop counters `i`/`j`/`k` and `*args`/`**kwargs` are allowed.
  *Verify by:* the naming convention is documented in `CONTEXT.md` and `.claude/CLAUDE.md`; PR review applies it. A grep-spot-check across `mash/**` returns zero hits for the canonical offender list (`\bext\b`, `\bdest\b`, `\bcmd\b`, `\bsrc\b`, `\bdisambig\b`, `\bdir\b` as identifiers — not in strings). *(manual on grep ambiguities.)*

- **AC-3 — Exceptions over sentinel returns** — Alternate / failure flows in `mash/flows/**` and `mash/helpers/selection/**` use `try`/`except`, not `if X is None: return …` or `if kind == "cancelled": …` checks on sentinel returns.
  *Verify by:* `scripts/check_conventions.py` AST-scans flow + selector classes and reports zero `return None` / `return ("cancelled", None)`-shape statements (outside the documented per-flow `except UserCancelled: return None` boundary); exits 0.

- **AC-4 — Descriptive exception names** — Every `except X` in `mash/**` (outside the top-level `MashCLI.run` entry point) names a specific, self-explanatory class. No bare `except`, no `except Exception`, no `except MashError` (the base) inside flows/helpers.
  *Verify by:* grep + manual review of every `except` clause in `mash/**`; each names one of: `UserCancelled`, `SourceNotFound`, `DestinationNotFound`, `LLMUnavailable`, `InvalidExtension` (or a successor added during the plan). The only `except MashError` lives in `MashCLI.run` as the top-level safety net.

- **AC-5 — Automated enforcement (OOP + sentinel) + documented naming** — A single script (`scripts/check_conventions.py`) runs AC-1 and AC-3 checks. Wired as a git pre-commit hook and runnable manually. Naming convention (AC-2) is enforced via docs + review, not script.
  *Verify by:* `python scripts/check_conventions.py` exits 0 on the refactored tree; a planted top-level `def` or planted `return None` in a flow makes it exit non-zero. `.git/hooks/pre-commit` calls the script. Naming convention text is present in `CONTEXT.md` and `.claude/CLAUDE.md`.

- **AC-6 — Identical CLI behavior (asserted by integration tests)** — Running `mash` from the CLI produces identical user-visible behavior to before this issue — same prompts, confirmations, file operations, output text. Asserted via integration tests rather than a separate baseline diff.
  *Verify by:* `tests/integration/` drives `mash --dry-run --yes <query>` for every intent (move, copy, rename same-ext, rename different-ext, delete file, delete folder, create file with ext, create file without ext, create folder, list, open, cat, chmod). Each test asserts the generated command + visible output text byte-equal a golden expected output captured pre-refactor on `main`. `pytest tests/integration/` exits 0.

- **AC-7 — Unit tests rewritten to fit new class structure, 100% branch coverage** — Tests under `tests/` are rewritten against the new class API and exercise every branch of every flow, helper, parser, picker, and selector.
  *Verify by:* `pytest tests/` exits 0; `pytest --cov=mash --cov-branch --cov-report=term-missing tests/` reports 100% branch coverage on `mash/**` with exclusions limited to: I/O boundaries (`subprocess.run` calls, `ollama.chat` calls), `if __name__ == "__main__"` blocks. Exclusions listed in `pyproject.toml` under `[tool.coverage.run] exclude_also`.

- **AC-8 — Entry-point intact** — The `.venv/bin/mash` `sys.path.insert` fix remains intact and `mash` still launches correctly.
  *Verify by:* `grep -q "sys.path.insert" .venv/bin/mash` succeeds AND `mash --dry-run --yes list` exits 0.

- **AC-9 — No leaked tracebacks** — Exceptions raised for alternate flows are caught and surfaced as the same friendly CLI messages — no raw tracebacks reach the user. The top-level `MashCLI.run` catches any uncaught `MashError` and prints a friendly message.
  *Verify by:* integration tests that simulate cancel at each cancel site (source picker, dest picker, extension picker, file-vs-folder, run confirm) assert stdout/stderr contains `Cancelled.` and zero `Traceback` lines. LLM-failure simulation asserts the existing `"LLM not working, please try again"` message and zero traceback lines.

- **AC-10 — Performance unchanged** *(manual)* — Cold-start time, time-to-first-prompt, and end-to-end latency are not noticeably worse than before.
  *Verify by:* user runs `time mash --dry-run --yes list` before and after; informal comparison confirms no noticeable regression (target: within ±20%).

**Decisions:**

- **AD-1 — One class per file (strict OOP), file name matches class name** — Every `.py` under `mash/**` (excluding `__init__.py`) defines exactly one class that owns all behavior and values for the file. File name is the snake_case form of the class name. Implements AC-1, AC-5. *(Principle: high cohesion / encapsulation. Convention: project-wide rule from issue #3.)*
- **AD-2 — Layered dependency direction** — Import layers (bottom to top): `exceptions/` → `intent.py` + `helpers/files/` + `helpers/commands/` (peer layer) → `helpers/console.py` + `helpers/llm_client.py` → `helpers/selection/` → `flows/` → `mash_cli.py`. Each layer imports only from layers below. No cycles. *Rationale:* makes the dependency graph trivially inspectable, prevents flows from accidentally re-importing each other. *(Principle: low coupling, layered architecture.)*
- **AD-3 — Static for stateless, instance for stateful** — Stateless classes (`Intent`, `Extensions`, `PathResolver`, `DirectoryContext`, `CommandSanitizer`, `CommandTemplates`, `CommandReference`) expose `@classmethod` / `@staticmethod` APIs and are never instantiated. Stateful classes (`Console`, `LLMClient`, `Disambiguator`, every `*Selector`, every `*Picker`, every `*Flow`, `MashCLI`) are instantiated and receive collaborators via constructor injection. *Rationale:* DI on stateless ops adds wiring without buying testability; reserving instance shape for things that hold state keeps the call graph honest. *(Principle: minimal indirection, DI where it pays.)*
- **AD-4 — Exception package, one file per class** — `mash/exceptions/<name>.py` for each exception. `__init__.py` re-exports the lot via assignment. Base `MashError` exists only for the top-level safety net in `MashCLI.run`; flows/helpers catch specific subclasses. Implements AC-3, AC-4. *(Convention: matches AD-1.)*
- **AD-5 — Single `UserCancelled` for all cancel paths** — Every cancel site (picker Enter, dest cancel, ext picker cancel, file-vs-folder cancel, run confirm decline) raises `UserCancelled`. Each `*Flow` catches it once at the boundary and prints `"Cancelled."`. *Rationale:* current behavior treats all cancels identically; splitting per-site would only multiply identical `except` branches. Implements AC-3, AC-9.
- **AD-6 — Shared pickers `CreateKindPicker` + `ExtensionPicker`** — Extracted from `create.py`; `RenameFlow` reuses `ExtensionPicker` (today duplicates the logic inline). *Rationale:* single-responsibility classes, eliminates the rename/create duplicate. *(Principle: DRY, single responsibility.)*
- **AD-7 — `list_flow.py` split four ways** — `ListFlow` (orchestrator) + `BrowseMode` (ls-and-menu loop) + `ActionMenu` (per-entry action picker) + `ListScope` (scoped context / list_entries / scope_label). *Rationale:* 609 LOC monolith violates cohesion; split makes each file navigable. *(Principle: single responsibility, file-shape predictability.)*
- **AD-8 — Naming convention via docs, not script** — Naming convention (no abbreviations, full descriptive words) lives as prose in `CONTEXT.md` + `.claude/CLAUDE.md` and is enforced via PR review. `scripts/check_conventions.py` does *not* lint names. *Rationale:* user-elected — name quality is judgment-laden and a strict abbreviation list creates more false positives than catches. Implements AC-2, AC-5.
- **AD-9 — `scripts/check_conventions.py` + git pre-commit hook** — Single Python script runs two AST checks: (1) no module-level `def`, (2) no `return None` / sentinel-tuple return in `*Flow` and `*Selector` classes. Wired as `.git/hooks/pre-commit`; runnable manually. Implements AC-5. *(No new dependencies; no CI workflow.)*
- **AD-10 — Integration tests subsume the baseline-diff regression net** — `tests/integration/` drives the CLI via `mash --dry-run --yes <query>` per intent and asserts byte-equal match against committed golden output captured pre-refactor on `main`. Replaces plan 4.0's loose `behaviour_baseline.sh` script. Implements AC-6.
- **AD-11 — 100% branch coverage with explicit exclusions** — `pyproject.toml` `[tool.coverage.run] exclude_also` lists: `subprocess.run` lines, `ollama.chat` lines, `if __name__ == "__main__"` blocks. Every other branch must be covered. Implements AC-7.
- **AD-12 — Top-level catch in `MashCLI.run`** — `MashCLI.run` catches `MashError` (base) as a safety net and prints a friendly message; flows catch only specific named subclasses. No bare `except` anywhere. Implements AC-4, AC-9.
- **AD-13 — Entry point `mash = "mash.mash_cli:MashCLI.main"`** — `pyproject.toml` script entry calls a `@classmethod main` on `MashCLI` that does `cls().run()`. Avoids a module-level `def` shim (which would violate AC-1). Implements AC-1, AC-8.
- **AD-14 — `RenameExtensionReconciler` class in `helpers/selection/`** — Dedicated class for the "keep old extension vs use new extension" prompt that fires when `Extensions.reconcile_rename(before, after_token)` returns a non-None `prompt_kind` (`"different_extension"` or `"add_extension"`). One file, one class, constructor takes `Console` (DI per AD-3). Public method `reconcile(before_path: str, after_token: str, before_extension: str, after_extension: str, kind: str) -> str` returns the final filename string; raises `UserCancelled` on Enter/decline. Shared by `RenameFlow` (Phase 5 G2) and `ActionMenu` (Phase 5 G6 rename action) — eliminates the inline reconciler duplicate that lives in both `rename.py` and `list.py` today. *Rationale:* `ExtensionPicker.pick()` returns a fresh extension from `EXTENSION_MAP`; reconciliation is a distinct UI (binary "keep vs change" choice) and warrants its own class per SRP. *(Principle: single responsibility, DRY.)*
- **AD-15 — Re-entry callback `run_with_args_fn` injected into flows that recover from not-found** — `OpenCatFlow`, `ListFlow`, and `ActionMenu` accept `run_with_args_fn: Callable[[list[str]], None]` via constructor. Used by the not-found menu (open/cat) and the list action menu when the user types a fresh free-text query instead of selecting an entry; the flow re-enters `MashCLI.run` with the typed query as new arguments. *Rationale:* preserves today's recovery behavior verbatim without coupling flows to `MashCLI` (the callback is wired by `MashCLI` in Phase 6 to point at `self.run_with_args`). Implements behavior preservation per AC-6.

**Design principles in play:** high cohesion (one class per file), low coupling (layered imports + DI for stateful collaborators), encapsulation (values live inside the class), single responsibility (picker/scope/menu split), DRY (shared `ExtensionPicker`).

**Patterns:** Layered architecture (AD-2). Dependency Injection for stateful collaborators (AD-3). Strategy-shaped per-verb Flow classes dispatched from `MashCLI` (existing, preserved). **Deliberately avoided:** abstract base classes for Flow / Selector (no behavior to share — concrete classes plus a documented dispatcher is simpler); registry/factory pattern for verb dispatch (a `match`/`if` chain in `MashCLI.run` is ~12 lines and beats a class registry).

## Claude Instructions

Context and constraints that help Claude implement this plan correctly:

- **Architecture binding:** Do not introduce structural choices (new modules, new patterns, new dependency directions, new cross-layer dependencies) not covered by `## Architecture decisions` above. If a task requires one, stop and surface it to the user before writing code — never improvise structure. Cite the relevant `AD-N` when a task implements a decision.
- **One class per file, file name matches class name:** every `.py` under `mash/**` (excluding `__init__.py`) defines exactly one class. File name is the snake_case form of the class name (per AD-1). Module constants, regex patterns, and the like live as class attributes — not at module level.
- **Naming convention:** All identifiers in `mash/**` use full descriptive words. No abbreviations — even one-letter shortenings (`ext`→`extension`, `dest`→`destination`, `cmd`→`command`, `src`→`source`, `disambig`→`disambiguator`, `dir`→`directory`, `args`→`arguments` *unless used in `*args`/`**kwargs`*, loop counters `i`/`j`/`k` allowed). PEP 8 for method/class names. A reader unfamiliar with the file should understand each name without surrounding context. Enforced by docs + review per AD-8.
- **Exceptions over sentinels:** No `return None` / `return ("cancelled", None)` style alternate flows in `*Flow` or `*Selector` classes. Raise a specific named exception (`UserCancelled`, `SourceNotFound`, `DestinationNotFound`, `LLMUnavailable`, `InvalidExtension`). The **only** sentinel `return None` allowed is the per-flow `except UserCancelled: print("Cancelled."); return None` boundary handler — and that one site is whitelisted by `scripts/check_conventions.py`.
- **Exception catching:** Flows catch only specific named subclasses. `MashError` is caught only at the top-level `MashCLI.run` safety net (per AD-12). No bare `except`, no `except Exception` anywhere in `mash/**`.
- **Layered imports:** `exceptions/` → `intent.py` + `helpers/files/` + `helpers/commands/` → `helpers/console.py` + `helpers/llm_client.py` → `helpers/selection/` → `flows/` → `mash_cli.py`. Each layer imports only from layers below. No cycles (per AD-2).
- **DI shape:** Stateful classes take collaborators via `__init__` (per AD-3). Stateless classes use `@classmethod` / `@staticmethod` and are never instantiated. Do not add factory classes or registries — `MashCLI._dispatch` is a `match` on `intent.verb`.
- **Test-rewrite scope:** AC-7 supersedes the original "tests pass unchanged" — tests get rewritten to fit the new class API and must cover every branch (100% branch coverage with documented exclusions per AD-11). Add tests for every new class and every branch of every flow / picker / selector.
- **CLI behaviour byte-identical:** All prompt text, menu wording, confirmation phrasing, and output strings must match today. Integration tests in `tests/integration/` assert this byte-equal against golden output captured in Phase 0 on current `main`.
- **Entry-point fragility:** The `.venv/bin/mash` `sys.path.insert` fix must survive Phase 6 (per AC-8). If `pip install -e .` rewrites the script, re-apply the fix manually per `.claude/CLAUDE.md`. Never use `.pth` files.
- **No leaked tracebacks:** Every exception path either gets caught at the flow boundary (`UserCancelled`, `InvalidExtension`) or by `MashCLI.run` (`MashError`, `LLMUnavailable`). User-visible output stays friendly (`"Cancelled."`, `"LLM not working, please try again"`).
- **Constraints:** Do not add new features. Do not change prompt text, menu wording, or any user-visible string. Do not add docstrings. Do not add dependencies (no `pre-commit` package, no linter, no CI workflow). Do not touch anything outside `mash/**`, `tests/**`, `scripts/**`, `CONTEXT.md`, `.claude/CLAUDE.md`, `pyproject.toml`, `.git/hooks/pre-commit`.
