# 2.1 — Mash Headless Stress Test

Run a full headless test suite of `mash` inside `Claude_testing/` and produce a bug report.

**Goal:** Identify bugs in command generation, fuzzy resolution, flag behaviour, and live execution. All tests stay inside `Claude_testing/`. No source files are edited.

**Status legend:**  ⬜ Not started · 🟡 In progress · ✅ Done

---

## Constants

```
MASH=/Users/max/Documents/GitHub/Mash\ agent/.venv/bin/mash
DIR=/Users/max/Documents/GitHub/Mash\ agent/Claude_testing
```

Every mash invocation uses:
```
cd "$DIR" && "$MASH" <flags> <prompt>
```

---

## Phase 0 — Fixture setup

Create a known file tree directly with shell commands (not via mash).

| Task | Exact command | Status |
|------|--------------|--------|
| Create dirs | `mkdir -p "$DIR/docs" "$DIR/reports" "$DIR/archive" "$DIR/tmp"` | ⬜ |
| Create files | `touch "$DIR/docs/overview.md" "$DIR/docs/notes.txt" "$DIR/reports/summary.pdf" "$DIR/main.py" "$DIR/config.json" "$DIR/data.csv"` | ⬜ |
| Verify tree | `find "$DIR" -not -path "*/.*"` | ⬜ |

No filesystem changes. Capture the `Run: ...` line from each invocation and assess correctness.

| # | Prompt | Expected command | Status |

| 5 | `--dry-run list files in docs` | `ls -la ./docs` | ⬜ |

| 7 | `--dry-run create a new file called readme.txt` | `touch ./readme.txt` | ⬜ |

## Phase 2 — Live execution

Use `--yes`. Verify filesystem state with `find` after each command.

| # | Prompt | Verify | Status |
|---|--------|--------|--------|
| 1 | `--yes create a directory called sandbox` | `sandbox/` exists | ⬜ |
| 2 | `--yes create a file called hello.txt inside sandbox` | `sandbox/hello.txt` exists | ⬜ |
| 3 | `--yes move hello.txt from sandbox to archive` | `archive/hello.txt` exists, `sandbox/hello.txt` gone | ⬜ |

## Phase 5 — Bug report

Write findings to `Claude_testing/stress_test_report.md`:
- Each bug: description, exact invocation, full output, expected behaviour

- Summary table: pass / fail counts

---

## Claude Instructions

- **Scope:** Read source files in `mash/` for context. Never edit them.
- **Working directory:** Every `mash` invocation must `cd` into `Claude_testing/` first.
- **Denied permissions:** If a tool call is denied, skip that test, mark it ⬜ SKIPPED, and continue.
- **Bugs to watch:**
  - False fuzzy match: `[debug] resolved:` shows an unrelated file
  - Intent label output: LLM outputs `move` instead of `mv`
  - Wrong destination: `mv ./data.csv ./reports/summary.pdf` instead of `mv ./data.csv ./reports/`
- **Do not fix bugs** — only document them.
